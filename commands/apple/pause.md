# Pause Work

Create comprehensive handoff documentation so you (or someone else) can resume exactly where you left off. This captures context that would otherwise be lost between sessions.

## Step 1: Gather Current State

Read all planning files:

```
Read: .planning/STATE.md
Read: .planning/PLAN.md
Read: .planning/APP.md
Read: .planning/ROADMAP.md
Read: .planning/VERIFICATION.md (if exists)
```

## Step 2: Identify In-Progress Work

Find what's currently being worked on:

1. **Current task** from PLAN.md (marked as in_progress)
2. **Recent file changes:**
   ```bash
   git status
   git diff --name-only HEAD~5
   ```
3. **Uncommitted work:**
   ```bash
   git diff --stat
   ```

## Step 3: Capture Context

Gather context that would be lost:

1. **What was being worked on:**
   - Current task description
   - Files being modified
   - Approach being taken

2. **What's partially complete:**
   - Code written but not finished
   - Tests written but not passing
   - Features started but not integrated

3. **What was learned:**
   - Discoveries during implementation
   - Problems encountered
   - Solutions found

4. **What's blocking (if anything):**
   - Technical blockers
   - Questions needing answers
   - Dependencies waiting

5. **What's next:**
   - Immediate next step
   - Remaining work for current task
   - Upcoming tasks

## Step 4: Create Handoff Document

Create `.planning/HANDOFF.md`:

```markdown
# Handoff Document

**Paused:** [timestamp]
**Phase:** [N] - [Phase Name]
**Task:** [Current task description]

---

## Current State

### What I Was Working On

[Detailed description of current work]

**Files being modified:**
- `path/to/file1.swift` - [what's being done]
- `path/to/file2.swift` - [what's being done]

**Approach:**
[Explanation of the approach being taken]

### Progress

**Completed in this session:**
- [x] [Thing 1]
- [x] [Thing 2]

**Partially complete:**
- [ ] [Thing 3] - [how far along, what remains]

**Not started:**
- [ ] [Thing 4]

### Code State

**Uncommitted changes:**
```
[git diff --stat output]
```

**Recent commits:**
```
[git log --oneline -5 output]
```

---

## Context & Learnings

### Key Discoveries

1. [Important thing learned during implementation]
2. [Another discovery]

### Problems Encountered

1. **[Problem]:** [Description]
   - **Status:** [Solved/Unsolved]
   - **Solution/Workaround:** [If solved]

### Technical Decisions Made

1. **Decision:** [What was decided]
   **Reason:** [Why]

---

## Blockers

### Active Blockers

- [ ] **[Blocker 1]:** [Description]
  - Needed: [What's needed to unblock]

### Questions Needing Answers

- [ ] [Question 1]
- [ ] [Question 2]

---

## Resume Instructions

### Immediate Next Step

[Specific, actionable next step]

**File to open:** `path/to/file.swift`
**Line to look at:** [line number or function name]
**What to do:** [Specific action]

### Remaining Work for Current Task

1. [Step 1]
2. [Step 2]
3. [Step 3]

### After Current Task

Next task: [Description from PLAN.md]

---

## Quick Resume Checklist

When resuming, do this:

1. [ ] Read this handoff document
2. [ ] Run `git status` to see uncommitted changes
3. [ ] Open [primary file] and go to [location]
4. [ ] Run `/apple:resume` to restore context
5. [ ] Continue with: [immediate next step]

---

*Generated by SwiftShip /apple:pause*
*Timestamp: [ISO timestamp]*
```

## Step 5: Update STATE.md

Add pause information to STATE.md:

```markdown
## Session Status

**Status:** PAUSED
**Paused At:** [timestamp]
**Handoff:** See .planning/HANDOFF.md
```

## Step 6: Completion Message

```
Work paused successfully.

Created: .planning/HANDOFF.md

Current state:
- Phase: [N] - [Name]
- Task: [Current task]
- Progress: [X]% complete

To resume later:
1. Run /apple:resume to restore context
2. Or read .planning/HANDOFF.md manually

Tip: Commit your work before closing:
  git add -A && git commit -m "WIP: [current task]"
```

## Tips for Good Handoffs

1. **Be specific** - "Working on line 45 of HealthKitManager.swift" not "Working on health stuff"
2. **Explain the why** - Future you won't remember the reasoning
3. **List blockers clearly** - Don't hide problems
4. **Make resume actionable** - "Open X, do Y" not "Continue working"
