# Release Scope Template

Copy this file to your project's `.planning/RELEASE.md`. It is the scope manifest
for one existing-app release — the single place that unifies the features and the
bug fixes shipping together. Generated by `/apple:release`.

```xml
<release version="[NEW_VERSION]" previous="[CURRENT_VERSION]" type="patch|minor|major" platform="iOS|macOS|both">
  <summary>[One or two sentences: what this update delivers and why.]</summary>

  <baseline>
    <!-- What already exists, from /apple:map's CODEBASE.md. Grounds the plan so
         it doesn't re-build shipped work. -->
    <shipped-version>[CURRENT_VERSION]</shipped-version>
    <architecture>[Detected pattern — MVVM / @Observable / SwiftData, etc.]</architecture>
    <test-coverage>[Rough % or "none" — drives how much regression safety to add]</test-coverage>
  </baseline>

  <features>
    <!-- priority: must | should | could. Omit this block for a pure bugfix release. -->
    <feature id="FT1" priority="must">
      <what>[The capability being added]</what>
      <touches>[Screens / models / services affected]</touches>
    </feature>
  </features>

  <bugs>
    <!-- severity: crash | high | medium | low. Omit this block for a pure feature release. -->
    <bug id="BG1" severity="crash" source="crash log|review|report|internal">
      <symptom>[What the user sees]</symptom>
      <suspected-area>[File/feature if known — else "unknown, investigate"]</suspected-area>
      <regression-test>required</regression-test>
    </bug>
  </bugs>

  <improvements>
    <!-- Optional. Pull from CODEBASE.md Technical Debt / Areas for Improvement. -->
    <improvement id="IM1">[Cleanup worth doing while in this area]</improvement>
  </improvements>

  <regression-scope>
    <!-- Existing flows adjacent to the changes above that MUST still work after
         this release. /apple:walkthrough drives these; they are the safety net a
         greenfield plan never captures for an existing app. -->
    <flow>[Existing flow 1 that touches the same screens/data]</flow>
    <flow>[Existing flow 2]</flow>
  </regression-scope>

  <out-of-scope>
    <item>[Explicitly NOT in this release]</item>
  </out-of-scope>

  <success-criteria>
    <criterion>All in-scope features and bugs complete</criterion>
    <criterion>Regression scope re-verified — no adjacent flow broke</criterion>
    <criterion>/apple:verify and /apple:review pass</criterion>
  </success-criteria>
</release>
```

## Key Sections

| Section | Purpose |
|---------|---------|
| `<baseline>` | What already exists (from `/apple:map`) — keeps the plan from re-building shipped work |
| `<features>` | New capabilities → become the **Build** phase tasks |
| `<bugs>` | Known defects → become the **Fix** phase tasks, each with a regression test |
| `<improvements>` | Optional tech-debt cleanup pulled from the codebase map |
| `<regression-scope>` | Existing flows to re-verify → drive the **Harden** phase / `/apple:walkthrough` |
| `<success-criteria>` | Definition of done for the release |

## How to Use

1. Run `/apple:release [scope]` to generate this file plus an intent-tagged `ROADMAP.md`
2. Run `/apple:plan [phase]` to expand a phase into tasks, then `/apple:build`
3. `/apple:milestone` archives this file when the release ships
